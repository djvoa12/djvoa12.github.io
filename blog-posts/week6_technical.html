Inheritance, as defined by Wikipedia, is when an object or class is based on another object or class, using the same implementation (inheriting from a class) or specifying implementation to maintain the same behavior (realizing an interface; inheriting behavior).

'Is-a' relationship is where one class A is a subclass of another class B (and so B is a superclass of A).

Composition, as defined by Wikipedia, is a technique by which classes may achieve polymorphic behavior and code reuse by containing other classes that implement the desired functionality instead of through inheritance.

'Has-a' relationship is where one object (often called the composited object) "belongs" to (is a part or member of) another object (called the composite type), and behaves according to the rules of ownership.

Composition is favored in a lot of ways over inheritance as it gives you more flexibility because it utilizes 'has-a' relationship. Inheritance may be a powerful tool, but code is often re-used and customized in ways not intended by the designer of the super class, which can lead to a mess. Moreover, it can be difficult to test classes that re-use code from a superclass. To get around the limitations of inheritance, the most obvious alternative is to use composition.

class Parent
 def implicit
 puts "Hello World!"
 end
 end

class Child < Parent
 end

dad = Parent.new
 son = Child.new

dad.implicit # => Hello world!
 son.implicit # => Hello world!

In the example above, inheritance is used to pass on the 'implicit' method to from the superclass 'Parent' to class 'Child'. The same result can be achieved by using composition with 'has-a' relationship. There is no hierarchy in the example below. We are initializing '@other' instance variable and assigning a new 'OtherClass' class. We can now define an 'implicit' method that applies the 'implicit' method that is pre-defined in 'OtherClass' class.

class OtherClass
 def implicit
 puts "Hello world!"
 end
 end

class Child
 def initialize
 @other = OtherClass.new
 end

def implicit
 @other.implicit
 end
 end

son = Child.new

son.implicit # => Hello world!

Another way to do composition is by using modules and mixins. This is done by creating a module with methods that are common to classes and then 'include' them in your class. See the example below.

module Other
 def implicit
 puts "Hello world!"
 end
 end

class Child
 include Other
 end

son = Child.new

son.implicit # => Hello world!
