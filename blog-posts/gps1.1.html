What is pair programming?

Pair programing occurs when two developers work together at one workstation. The driver types the code while the navigator reviews each line of code that is being typed in. They also share thoughts as they work through the process of coding and switch roles frequently.

Why is pair programming important to DBC?

At DBC, one is expected to pair almost all the time. This is practiced because pair or group programming is very common in the actual workplace, and DBC wants its Boots to become very familiar with this. It also helps us learn faster and make fewer mistakes.
