The position value in CSS helps optimize layouts and manipulating elements to be in you desired location.  There are six values associated with position property: static, relative, absolute, fixed, initial and inherit.

Static: Every position is static by default.  It resides in the normal page flow and the positional properties, such as left/right/top/bottom/z-index, have no effect.

Relative: It behaves same as static value, but the positional properties take effect.  This means you can move the element from the original (static) position in different directions based on the positional properties you give.



Fixed: It is positioned relative to the viewport and stays fixed in the same location, even if the page is scrolled.  The positional properties are used to specify the location.  Check out Fixed Position Example!  You will see that there is a div with fixed position at the bottom right-hand side of your browser.

Absolute: This is the trickiest position value.  It behaves like fixed value, except relative to the nearest positioned (i.e. anything but static value) ancestor, instead of relative to the viewport.  If there are no nearest positioned ancestors, it will use the document body for its relative base.



Inherit: It takes position value from its parent.

Initial: It sets the property to its default value.
