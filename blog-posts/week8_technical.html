Rails, often associated with Ruby on Rails, is an open source web application framework written in Ruby.  It utilizes Model-View-Controller (MVC) architecture to organize web application programming, and its design makes programming web applications easier by making assumptions about the needs of developers.  It allows one to write less code while achieving the same or even greater results when compared to many other languages and frameworks.

To better understand Rails, you need to know more about Model-View-Controller framework.  MVC is a way of organizing code to conceal complexity and facilitating the implementation of user interfaces.  It separates the software application into three interconnected sections: the model, the controller, and the view.  Ruby on Rails takes all three of these concepts and packages them together, and developers only need to install the rails gem to have this full development environment for writing web applications.

The Model

The Model handles all the database interaction in your web application.  It retrieves and stores all objects from the database and also manages data integrity issues.  For example, if you tried to enter a negative integer into a field that is not suppose to contain negative integers, the code in the Model would reject to implement this action.

Controller

The Controller handles all the web server interaction and assumes its role as the conduit between the Model and the View.  It can grab an object from the database through the Model and then push it to the View, which will render it.  Basically, it can send commands to both the Model and the Viewer to either change the Model’s state (i.e. editing a document) or the View’s presentation of the Model (i.e. scrolling through a document).

The View

The View creates the HTML code that is displayed in the browser by receiving the data from the Controller.  The View’s function is to focus on formatting and style, and it’s common to see the code being a mixture of HTML and Ruby.
